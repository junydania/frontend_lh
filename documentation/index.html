Your skill level

For installation "as is" you don’t need any knowledge of coding. However, customizing of template requires some skills in web programming:

  • for a simple customization you’d be required to have some knowledge of HTML, CSS and jQuery;
  • for advanced customization, in addition to the above, you’re required to have knowledge of Gulp, Handlebars and JavaScript.

Choose the version that matches your current technical capabilities: Basic or Developer.

Basic

Basic version allows you to configure the basic elements of the template in a simple way. You'll be required to have knowledge of html, css and jquery.### Getting Started

Ajax smooth loading works only on a web server. So first, you need to transfer the folder www from the template archive into the root directory of your website.

Specific actions in this phase depend on your hosting provider, so contact him for clarification.

File structure

  • css — folder with style files. Description.
  • favicons — folder with favicon images. See tips on replacement.
  • fonts — folder with fonts. Description.
  • images — folder with all images of the template.
  • js — folder with the scripts of the template. Description.
  • vendors — folder with vendor libraries that have sub-files with relative paths.

CSS Files used for this template

Styles are concatenated into a single file, but uncompressed and available for basic editing. Each new section is indicated by the comment.

JavaScript Files used for this template

Scripts are concatenated into 2 files, but uncompressed and available for basic editing. File critical.js contains critical scripts for displaying the first screen. Everything else, including vendor libraries, is presented in the file main.js.

Fonts

To add fonts, use the rule @font-face in the file main.css.

Favicon

To create your own icon, use the service favicongenerator. Use 2 versions, monochrome and color, for perfect displaying on all platforms. You can find the necessary templates in directory favicons/template. After generating the files, replace the contents of the folder favicons. You may also need to change the code after <!-- favicon--> in <head> section on each page of the template.

Developer

Developer version allows you to customize every element of the theme. It requires knowledge of Handlebars(HTML), SASS(CSS), JavaScript(jQuery) и Gulp.

Getting Started

This manual is intended for MacOS users, but approximately the same sequence remains for Windows and Linux users.

1) To install NodeJS visit website, download the package, according to your operating system and install it on your computer by following tips.

2) After installation enter the command npm -v in the terminal to find out NodeJS version

3) Task Manager Gulp and Bower utility will be installed on your computer globally like default programs to be used again and again in your projects. Write sudo npm i bower gulp -g in a terminal to install them. You will be required to confirm the administrator password. Please, note that the password characters are not displayed.

4) (Optional) Accept and install Command Line Developer Tool if it’s proposed.

5) Open the project folder, run terminal in it and run the command of template installation: npm i. After some time, about 30-40 seconds, depending on the Internet speed, all the necessary dependencies will be installed in the root directory.

6) Run the command of bower dependencies installation: bower i

7) After installing the plugin run the command gulp

File Structure

  • src/assets — favicons, fonts and images, some files are divided into subdirectories for convenience;
  • src/data — folder for data storing. You can create additional files for convenient structuring of information;
  • src/scripts — folder with scripts. All the files in the root directory will be compiled without concatenation;
  • src/styles — folder with styles. Detailed information;
  • src/views — folder with handlebars templates;
  • bower.json — manual for bower to add dependencies. Detailed information;
  • config.json — Gulp configuration file. Detailed information;
  • gulpfile.js — Gulp main file. Detailed information;
  • package.json — manual for Gulp to add dependencies. Detailed information;
  • vendors.json — configuration file to export vendor libraries "as is" from bower.

Gulp file review

  • dependencies — NodeJS dependencies;
  • paths — path variables;
  • templates — command for building handlebars templates;
  • SASS — command to build styles;
  • scripts — command to build scripts;
  • images — command for compression and exporting images;
  • fonts — command for exporting fonts;
  • favicons — command for exporting favicons;
  • vendors — command for exporting vendor libraries. Detailed information;
  • watcher — tracking browser changes and update;
  • server — server browserSync;
  • clean — command for cleaning up;
  • build — building template;
  • default — default command (gulp). Performs cleaning up, building the template, starting the server, tracking changes.

config.json

The configuration file provides a convenient way to transfer some settings to gulp. All values are of logical data type (Boolean), i.e. they can only be true and false.

prettify_html — auto-formatting html;

compress_html — minification html;

compress_js_css — compression JavaScript and CSS;

compress_img — image compression;

open_browser — opening browser after launching Gulp.

vendors.json

File allows you to assign the libraries that will be copied "as is" from the bower, with all its contents. Just add the path of the library in the appropriate file, using the following format: bower_components/font-awesome/**/*. This **/* means that you want to copy all sub-directories and files.

CSS Files used for this template

All files in the root directory will be compiled without concatenation.

partials

To add style files, you should edit the main file style.scss. For example, to add file fonts.scss, you should enter @import 'partials/fonts'. The file extension is optional.

bower_components

Here you can add styles of any library loaded with bower. To add .sass or .css file, enter its path in the format //=require slick-carousel/slick/slick.css relative to the folder bower_components.

vendors

To add vendor libraries that were copied "as is", use directive @import url(../vendors/font-awesome/css/font-awesome.min.css);.

JavaScript Files used for this template

All files in the root directory will be compiled without concatenation. By default, the template has 2 main files: critical.js and main.js. The first one is used for critical scripts that affect the display of the first screen and the second is used for all other scripts.

partials

To add script files, you should edit the main files critical.js и main.js. For example, to add file jquery.lazyload.js, you should enter //=require partials/jquery.lazyload.js.

bower_components

You can add the scripts in any library downloaded with bower. To add .js file, enter its path in the format //=require slick-carousel/slick/slick.css relative to the folder bower_components.

Fonts

To add fonts, use the rule @font-face в файле partials/fonts.scss.
